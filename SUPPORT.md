# How to Report an Issue

Found a bug or have a suggestion? Here's how to let us know — it only takes a minute.

## Before You Submit

1. **Check the in-app FAQ** — click the **?** icon in the app's titlebar. Many common issues (USB cables, device detection, LLM setup) are answered there.
2. **Search existing issues** — someone may have already reported it: [View Issues](https://github.com/superbasicstudio/fresh-mayhem-PUBLIC/issues)

## How to Submit a Bug Report

1. Go to **[New Issue](https://github.com/superbasicstudio/fresh-mayhem-PUBLIC/issues/new/choose)**
2. Click **"Bug Report"**
3. Fill in the form — it will ask for:
   - **Platform** — your OS (Ubuntu, Fedora, macOS, Windows, etc.)
   - **Device** — which hardware you're using (HackRF One, PortaPack, etc.)
   - **App version** — shown in the green BETA badge in the top-left of the app
   - **What happened** — describe what went wrong
   - **Steps to reproduce** — how can we recreate the problem?
4. **Optional but helpful:**
   - **Screenshots** — drag and drop images directly into the form
   - **Diagnostic logs** — in the app, go to Settings → Diagnostics → Export Logs

## How to Request a Feature

1. Go to **[New Issue](https://github.com/superbasicstudio/fresh-mayhem-PUBLIC/issues/new/choose)**
2. Click **"Feature Request"**
3. Describe what you'd like and how it would help your workflow

## Don't Have a GitHub Account?

You'll need a free GitHub account to submit issues. Sign up at [github.com/signup](https://github.com/signup) — it takes 30 seconds.

If you'd rather not create an account, you can reach us at **hello@superbasic.studio**.
